1. 기반 구축

완료
  • DB 스키마 마이그레이션
    PHP CodeIgniter 3 참조 SQL을 분석해 Next.js 프로젝트용 스키마로 재설계.
    게시물과 댓글을 board_posts 단일 테이블로 통합. parent_id / thread_id / depth 컬럼으로 트리 구조 표현. 소프트 삭제(is_deleted), 이미지 저장용 attachments 테이블 포함.
    테이블역할주요 컬럼
    users회원 정보id, username, display_name, password_hash
    user_sessions세션 관리session_id, user_id, is_active, logout_at
    board_posts게시물 + 댓글 통합parent_id, thread_id, depth, is_deleted
    attachments이미지 저장post_id, mime_type, byte_size, data(LONGBLOB)
    • migrations/schema.sql
    • lib/types.ts
    • lib/db.ts
  • 인증 시스템 구현
    로그인·로그아웃·회원가입 API 라우트 및 UI 구현. 비밀번호 해싱은 PBKDF2(Node crypto) 사용. 세션은 node_board_session httpOnly 쿠키로 관리. 로그인·가입 성공 알림 메시지 제거 — 오류 메시지만 표시.
    • app/api/auth/*
    • lib/auth.ts
    • app/page.tsx

2. 게시판 기능 구현

완료
  • 게시물 수정·삭제 버튼
    상세 화면에 목록 / 수정 / 삭제 순서로 버튼 배치. 본인 게시물에만 수정·삭제 버튼 노출. 관리자는 모든 게시물 삭제 가능.
    • app/page.tsx
    • app/api/posts/[id]/route.ts
  • 댓글·답글 답글/수정/삭제 버튼
    모든 depth의 댓글에 답글 / 수정 / 삭제 버튼 추가. ReplyItem 컴포넌트가 재귀 렌더링하므로 모든 중첩 레벨에 자동 적용. 본인 댓글에만 수정·삭제 노출, 로그인 유저에게만 답글 노출.
    • app/page.tsx
    • app/globals.css
  • BOARD 로고 클릭 시 목록 이동
    헤더의 BOARD 텍스트를 <button>으로 변경. 클릭 시 selectedPost, 작성 폼, 수정 폼 등 전체 상태 초기화 후 목록으로 복귀.
    • app/page.tsx
    • app/globals.css
  • 답글 폼 중복 노출 수정 / 컨텍스트 레이블
    답글 버튼 클릭 시 인라인 폼과 하단 폼이 동시에 열리던 문제 수정. openReplyId !== null이거나 editingReplyId !== null이면 하단 기본 폼 숨김. 폼 상단에 현재 작성 대상("원글에 댓글" / "닉네임 님에게 답글") 표시.
    • app/page.tsx
    • app/globals.css
  • 게시판 검색 기능
    목록 화면 상단에 검색 바 추가. 분류 / 제목 / 작성자 탭 선택 후 검색.
    제목·작성자: 텍스트 부분 일치(대소문자 무관).
    분류: 전체 / 공지 / 일반 드롭다운 선택.
    검색 중 결과 건수 표시 및 초기화 버튼 노출. 모두 클라이언트 필터링(추가 API 없음).
    • app/page.tsx
    • app/globals.css

3. 이미지 첨부 기능

완료
  • 이미지 업로드 및 DB 저장
    게시물·댓글·답글에 jpg/png/gif 이미지 첨부 기능 추가. 이미지를 파일 시스템이 아닌 MySQL LONGBLOB으로 직접 저장. 파일당 최대 8 MB, 허용 MIME 타입 검증(validateImageFiles). GET /api/attachments/[id]로 서빙, Cache-Control: private, max-age=86400 적용.

    • app/api/posts/route.ts

    텍스트 없이 이미지만 첨부해도 댓글·답글 등록 가능. 프론트: required={draft.files.length === 0}으로 조건부 필수. 백엔드: if (!content && images.length === 0) 조건으로 검증.

8. 트러블슈팅

버그 수정
  • 어드민 비밀번호 해싱 불일치
    스키마 시드에 CI3 bcrypt 해시가 포함되어 있어 로그인 실패.
    CI3는 bcrypt 사용, Next.js 인증은 PBKDF2만 검증 → 항상 로그인 실패
    hashPassword("123456")로 PBKDF2 해시 생성 후
    UPDATE users SET password_hash='pbkdf2$...' WHERE username='admin' 실행
    • lib/auth.ts
    • migrations/schema.sql
  • TypeScript: Buffer → BodyInit 타입 오류
    이미지 서빙 라우트에서 mysql2가 반환하는 Buffer를 NextResponse에 전달 시 타입 오류.
    Buffer는 BodyInit에 직접 할당 불가 — TS 컴파일 오류
    row.data as unknown as BodyInit 캐스팅으로 해결
    • app/api/attachments/[id]/route.ts
  • MySQL max_allowed_packet 초과
    XAMPP 기본값 1 MB로는 이미지 INSERT 불가.
    ER_NET_PACKET_TOO_LARGE — 8 MB 이미지 저장 실패
    런타임: SET GLOBAL max_allowed_packet = 67108864;
    영구 적용: my.ini에 max_allowed_packet=64M 추가
    • XAMPP my.ini
  • POST /api/posts 404 + "Failed to find Server Action"
    multipart 라우트 핸들러 추가 후 .next 캐시가 오래된 Server Action 정보를 참조.
    .next/ 빌드 캐시가 새 Route Handler를 인식하지 못하고 Server Action으로 오해
    .next/ 디렉터리 삭제 후 npm run dev 재시작으로 해결
    • .next/ (캐시)
  • CSS가 중첩 댓글 버튼 숨김
    depth 2 이상 댓글의 답글/수정/삭제 버튼이 보이지 않던 문제.
    .reply-list .reply-button { display: none } 규칙이 잔존해 중첩 버튼 전체 숨김
    해당 CSS 규칙 삭제로 해결 — 모든 depth에서 버튼 정상 노출
    • app/globals.css
  • ER_BAD_FIELD_ERROR: Unknown column 'p.is_notice'
    관리자 대시보드 기능 추가 후 게시판 접근 시 500 오류 발생.
    02_admin_mypage.sql 마이그레이션 미실행 — is_notice, is_banned, reports, blocked_words 전부 없음
    phpMyAdmin 없이 XAMPP MySQL CLI로 직접 실행:
    "C:\xampp\mysql\bin\mysql.exe" -u root node_board < migrations/02_admin_mypage.sql
    • migrations/02_admin_mypage.sql
  • TS6133: handleReport 선언은 되었지만 읽히지 않음
    신고 API 함수를 추가했으나 JSX에서 사용하지 않아 TypeScript 빌드 오류.
    'handleReport'이(가) 선언은 되었지만 해당 값이 읽히지는 않았습니다.
    게시물 상세 액션 버튼 영역과 ReplyItem 댓글 액션 영역에 신고 버튼 JSX 추가.
    onReport prop을 ReplyItemProps에 포함하고 호출부에서 전달.
    • app/page.tsx
  • comment_likes 테이블 없음 — 좋아요 API 500 오류
    댓글 좋아요 기능 구현 후 개발 서버 재시작 시 INSERT 쿼리 실패.
    Table 'node_board.comment_likes' doesn't exist — schema.sql은 전체 재생성용이라 기존 DB에 새 테이블이 없음
    lib/db.ts 풀 초기화 블록에 CREATE TABLE IF NOT EXISTS comment_likes ... 추가.
    앱 최초 요청 시 자동 생성 — 이후 테이블이 있으면 무시(idempotent).
    globalForMysql.dbInited 플래그로 중복 실행 방지.
    • lib/db.ts
    • migrations/schema.sql
  • Cannot find module './276.js' (.next 캐시 파편화)
    파일 변경 후 개발 서버 재시작 없이 빌드만 반복하면 webpack 청크 번호가 달라져 런타임 오류 발생.
    webpack-runtime.js가 존재하지 않는 청크 파일(276.js)을 참조 — 페이지 렌더링 500
    Remove-Item -Recurse -Force .next 후 npm run build 재실행으로 해결
    • .next/ (캐시)

14. 삭제 댓글 플레이스홀더

신규
  • 삭제된 댓글을 트리에서 제거하지 않고 자리 표시
    기존에는 댓글 조회 쿼리에 AND is_deleted = 0 조건이 있어 삭제된 댓글이 아예 제외됐음.
    삭제된 댓글에 답글이 달려 있으면 하위 대화 흐름이 끊기는 문제가 있었음.

    변경 내용:
    • API: 댓글 쿼리에서 AND is_deleted = 0 제거 — 삭제된 댓글도 트리에 포함.
    • ReplyNode 타입에 isDeleted: boolean 필드 추가, toReplyNode()에서 세팅.
    • replyCount는 replyRows.filter(r => r.is_deleted === 0).length로 비삭제 댓글만 카운트 유지.
    • 프론트 ReplyItem: reply.isDeleted이면 작성자·날짜·내용·버튼 전부 숨기고 "삭제된 댓글입니다." 텍스트(.comment-deleted)만 렌더링.
    • 하위 답글(reply.replies)은 삭제 여부와 무관하게 그대로 표시 — 대화 맥락 유지.
    • app/api/posts/[id]/route.ts

  `id` INT UNSIGNED NOT NULL AUTO_INCREMENT,
  `username` VARCHAR(30) NOT NULL,
  `display_name` VARCHAR(50) NOT NULL,
  `password_hash` VARCHAR(255) NOT NULL,
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),

  `session_id` VARCHAR(128) NOT NULL,
  `user_id` INT UNSIGNED NOT NULL,
  `username` VARCHAR(30) NOT NULL,
  `display_name` VARCHAR(50) NOT NULL,
  `ip_address` VARCHAR(45) NOT NULL,
  `user_agent` VARCHAR(255) NOT NULL,
  `login_at` DATETIME NOT NULL,
  `last_activity_at` DATETIME NOT NULL,
  `logout_at` DATETIME NULL,
  `is_active` TINYINT(1) NOT NULL DEFAULT 1,
  PRIMARY KEY (`session_id`),
  KEY `idx_user_sessions_user_id` (`user_id`),
  KEY `idx_user_sessions_active` (`is_active`, `last_activity_at`),
  CONSTRAINT `fk_user_sessions_user`
    FOREIGN KEY (`user_id`) REFERENCES `users` (`id`)
    ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
